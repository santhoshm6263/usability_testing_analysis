# usability_testing_analysis
# TaskFlow App — Usability Testing & Analysis Case Study

A complete UX case study focused on evaluating and improving the usability of **TaskFlow**, a mobile task management app prototype.

## 📌 Project Overview

This project documents the end-to-end usability testing process for **TaskFlow**, a Figma-based mobile prototype designed to help users manage tasks efficiently.

The study was conducted before development to validate key user flows, uncover usability issues, and recommend improvements based on real user behavior.

---

## 🎯 Goals

The primary objectives of this usability study were to:

- Identify friction points in core user journeys
- Measure task completion success
- Evaluate efficiency through time-on-task
- Collect qualitative feedback from users
- Improve the product before engineering handoff

---

## 📱 Product Tested

**TaskFlow** — Mobile task management app prototype

### Prototype Screens:

- Splash Screen
- Onboarding
- Login
- Home Dashboard
- Add Task
- Profile / Stats

---

## 👥 Participants

5 moderated participants matching the target audience:

- Age: 18–35
- Daily smartphone users
- Previously used to-do apps
- Non-designers

---

## 🧪 Testing Method

Each participant completed 3 realistic scenarios using the **think-aloud method**.

### Session Structure

1. Pre-session briefing  
2. Warm-up exercise  
3. Task scenarios  
4. Observation logging  
5. Post-test interview  
6. SUS survey

---

## 📋 Test Scenarios

### Scenario 1 — First Launch to Home Screen

Users opened the app, completed onboarding, and reached the home screen.

### Scenario 2 — Add a New Task

Users created a task with:

- Title
- Due date
- Priority level

### Scenario 3 — Complete Task + View Stats

Users marked a task complete and checked progress metrics in Profile.

## 🔍 Major Findings

### 1. Priority Selection Was Confusing

Most users did not understand what the colored priority dots meant.

**Impact:** Slow task creation and incomplete forms.

---

### 2. Floating Action Button Was Hard to Discover

Several users struggled to find the add-task button.

**Impact:** Increased task completion time.

---

### 3. Profile Icon Placement Was Unclear

Users expected Profile to appear on the far-right side of bottom navigation.

**Impact:** Wrong taps and slower navigation.

---

### 4. Stats Labels Were Ambiguous

Users were unsure whether “Tasks completed” referred to today or all-time totals.

**Impact:** Reduced trust and clarity.

---

## ✅ Recommended Improvements

### Priority UI

Replace color dots with labeled tags:

- Low
- Medium
- High
- Urgent

### Add Task CTA

- Increase FAB size
- Add “Add Task” label
- Use first-time pulse animation

### Navigation

Move Profile icon to rightmost tab.

### Stats Clarity

Change labels to:

- Tasks completed (all time)
- Completed today
